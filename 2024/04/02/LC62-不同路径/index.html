62.不同路径

一个机器人位于一个 m x n 网格的左上角 （起始点在下图中标记为 “Start” ）。

机器人每次只能向下或者向右移动一步。机器人试图达到网格的右下角（在下图中标记为 “Finish” ）。

问总共有多少条不同的路径？

示例 1：

输入： m = 3, n = 7
输出： 28

示例 2：

输入： m = 3, n = 2
输出： 3
解释：
从左上角开始，总共有 3 条路径可以到达右下角。

  1. 向右 -> 向下 -> 向下
  2. 向下 -> 向下 -> 向右
  3. 向下 -> 向右 -> 向下

示例 3：

输入： m = 7, n = 3
输出： 28

示例 4：

输入： m = 3, n = 3
输出： 6

提示：

  • 题目数据保证答案小于等于 

解法一(动态规划)

思路分析：

  1. 对于多少条路径问题，采用动规来解决
  2. 即思考动规五部曲：
    1. 确定dp数组及下标的含义；即dp[i][j]表示到达点(i, j)的路径总数
    2. 推导状态转移方程，根据题意，对于到达点(i, j)有两种方式，即从点(i-1, j)往下走一步，或从点(i, j-1)往右走一步，即到达点(i,j)的路径总数，为到达点(i-1, j)的路径总数与到达点(i, j-1)的路径总数 的和；即状态转移方程为dp[i][j] = dp[i-1][j] + dp[i][j-1]
    3. 对dp数组进行初始化；因为只能往右走或只能往下走，即第一行的点与第一列的点均只有一种方法
    4. 思考如何遍历dp数组；即对网格内的点按顺序遍历，因为得到点dp[i][j]的状态需要先得到在点(i, j)上面和左边的点
    5. 打印dp数组，与思路和题意相对应

思路分析：

  1. 根据解法一的状态转移方程可以得知，某个点的状态只与上一行同列的状态以及同行左边的状态有关；因此可以进行状态压缩，只用一维数组来进行推导
  2. 即从上往下遍历，然后再从左往右遍历，此时对于点(i, j)的状态dp[j] = dp[j] + dp[j-1]，即未更新的dp[j]表示的是上一行同列的状态，dp[j-1]则是表示同行左列的状态

解法三(组合数学)

思路分析：

  1. 即对于从点(1, 1)到达点(m, n)，一共需要往下走m-1步，往右走n-1步
  2. 即总共需要移动m+n-2次，那么在其中选择m-1步进行组合，即选择必须走的m-1步在总移动次数中所处于的位置，不同位置则产生不同路径；即到达终点的走法(路径数)为；带入m、n即可得到最终结果

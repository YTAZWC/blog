63.不同路径 II

一个机器人位于一个 m x n 网格的左上角 （起始点在下图中标记为 “Start” ）。

机器人每次只能向下或者向右移动一步。机器人试图达到网格的右下角（在下图中标记为 “Finish”）。

现在考虑网格中有障碍物。那么从左上角到右下角将会有多少条不同的路径？

网格中的障碍物和空位置分别用 1 和 0 来表示。

示例 1：

输入： obstacleGrid = [[0,0,0],[0,1,0],[0,0,0]]
输出： 2
解释： 3x3 网格的正中间有一个障碍物。
从左上角到右下角一共有 2 条不同的路径：

  1. 向右 -> 向右 -> 向下 -> 向下
  2. 向下 -> 向下 -> 向右 -> 向右

示例 2：

输入： obstacleGrid = [[0,1],[0,0]]
输出： 1

提示：

  • m == obstacleGrid.length
  • n == obstacleGrid[i].length
  • obstacleGrid[i][j] 为 0 或 1

解法一(动态规划)

思路分析：

  1. 对于该题可以先做LC 62.不同路径；同样使用动态规划算法进行求解
  2. 根据动规五步曲：
    1. 先确定dp数组以及下标的含义，即dp[i][j]表示到达点(i,j)的总路径数
    2. 然后推导状态转移方程式：因为网格内存在障碍物，即不能走存在障碍物的点，即不存在经过障碍点的路径，所有障碍点处的路径总数为0；由此可以得出状态转移方程为：dp[i][j] = obstacleGrid[i][j] == 1 ? 0 : dp[i-1][j] + dp[i][j-1]，即当前点位障碍点时，路径数直接为0，若当前点不是障碍点，则可有上一行的点向下走一步，或左列的点向右走一步；**对于点(i-1, j)是否为障碍点不需要判断，因为此时所处位置已经经过点(i-1, j)，若为障碍点，则上一行时已经做过判断，并设为dp[i-1][j] = 0**，对求当前点dp[i][j]不影响，对于点(i, j-1)同理
    3. 然后对dp数组进行初始化，因为只能往右或往下走，所以第一列和第一行中，所有点的路径总数最多为1，因为第一行和第一列中可能存在障碍物，若存在障碍物时，则不能继续沿着第一列往下走或沿着第一行往右走，即障碍物后续点的路径数只能为0
    4. 然后考虑对dp数组的遍历，因为只能往下或往右走，得到当前点的路径数需要依靠上一行和上一列的点，因此遍历顺序为：从上往下，再从左往右
    5. 将dp数组输出，检查是否符合题意和思路

解法二(状态压缩(滚动数组))

思路分析：

  1. 对于解法一得到的状态转移方程可以发现，点(i, j)的状态至于上一行同列的点以及左列同行的点有关，因此可以考虑只使用一维数组，滚动记录每一行的状态，直到最后一行；此时dp[n-1]即为到达右下角的总路径数
  2. 同时需要注意，初始化一维数组时，即对应网格中的第一行，依旧需要考虑障碍物对初始化的影响，若第一行中的某点为障碍物，则在这点之后的点均不可到达，路径总数均为0
  3. 同时需要考虑细节，因为初始遍历顺序依旧为从上往下，再从左往右，但是因为使用一维数组来表示每一行的状态，对于网格中的第一列，需要在遍历一维数组的过程中，时刻初始化，即需要判断每一行的第一个点，是否为障碍物，以及是否已经出现过障碍物，即对应二维dp中对第一列的初始化。
  4. 同时此时的状态转移方程变为dp[j] = obstacleGrid[i][j] == 1 ? 0: dp[j] + dp[j-1]；更新前的dp[j] 即为上一行同列的点
